# hj-components-rn

## 开发介绍

本地启动

```bash
yarn start
```

上传打 tag

```bash
yarn release
```

## commit 提交格式（注意冒号后面有空格）

常用的 type 类别

- build
- ci
- chore：构建过程或辅助工具的变动
- docs：文档（documentation）
- feat：新功能（feature）
- fix：修补 bug
- perf：性能优化
- refactor：重构（即不是新增功能，也不是修改 bug 的代码变动）
- revert：复原
- style： 格式（不影响代码运行的变动）
- test：增加测试
